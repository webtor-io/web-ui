9. Alerts & Progress Log

Progress alerts are JS-driven components (initProgressLog from assets/src/js/lib/progressLog.js) that render real-time log output. Container uses card-based styling (bg-w-surface, border-w-line) with colored dot prefixes (pulsing on in-progress) and tinted summary rows (no dots). See live demos at /tests/progress_log.

Log Line States

regular info line (gray dot, gray text)
in progress — pulsing cyan dot
status update — pulsing cyan dot + counter42
done — cyan checkmark ✓done
warning — pink warning ⚠warn
error — pink cross ✕error
...continued line (no dot, indented)
pre states: (none)=info, inprogress, statusupdate, done, warn, error, continue
Icons: gray dot (info) · cyan pulsing dot (inprogress/statusupdate) · cyan ✓ (done) · pink ✕ (error) · pink ⚠ (warn)
Status text via span.task-status — hidden on done/error/warn (icons are sufficient), shows (42) for statusupdate

Summary Rows

Tinted rows shown below a log line when done/warn/error includes a message. No dot prefix.

transfer complete
file saved successfully
slow connection
speed limited to 5Mbps — donate to increase!
peer disconnected
no active seeders found for this torrent
download ready
redirecting to resource page...
Cyan tint: done-summary / download-summary / redirect-summary — bg rgba(0,206,201,0.1), text #00cec9
Pink tint: warn-summary / error-summary — bg rgba(232,67,147,0.1), text #fd79a8
Summary rows use mx-5 rounded-lg to inset within the container

Block Variant with Close Buttons

progress-alert-block is the standard block layout used in templates. Close buttons appear via .alert-close-wrapper container, revealed by JS on close(). The .close class inside progress-alert inherits btn-soft-cyan styling.

Container: bg-w-surface (#111827) border border-w-line shadow-lg rounded-box popin
Log lines: font-mono text-sm text-w-sub with colored prefix icon via ::before pseudo-element
Icons: gray dot (info) · pulsing cyan dot (inprogress/statusupdate) · ✓ checkmark (done) · ✕ cross (error) · ⚠ warning (warn)
Close buttons: .close inside progress-alert = btn btn-sm with cyan soft styling (rgba(0,206,201,0.10) bg, rgba(0,206,201,0.30) border)
Entry animation: popin keyframes (scaleX 0.95→1, opacity 0→1)

JavaScript API

import { initProgressLog } from "../../lib/progressLog";

// Initialize on a container element (must have .log-target inside)
const log = initProgressLog(document.querySelector('.my-alert'));

// Chain log entries
log
  .info('regular info message')               // gray dot, gray text
  .inProgress('task1', 'connecting...')      // cyan pulsing dot
  .updateStatus(42)                          // updates to "(42)" format
  .done('download complete')                 // cyan checkmark + summary row
  .warn('speed limited')                     // pink warning icon + summary row
  .error('connection failed')                // pink cross icon + summary row
  .close();                                  // reveals .alert-close-wrapper
Import from assets/src/js/lib/progressLog.js
Each method returns an Entry for chaining · inProgress(tag, message) creates new line · done/warn/error(message?) updates the tagged line and optionally adds summary row

Template Pattern

<!-- Block variant (standard) -->
<form class="hidden progress-alert progress-alert-block closeable">
    <div class="log-target"></div>
    <div class="pt-3 flex justify-end hidden alert-close-wrapper">
        <div class="error hidden">
            <a class="close closeable-close">error action</a>
        </div>
        <a class="close closeable-close">got it!</a>
    </div>
</form>

<!-- One-line variant -->
<div class="progress-alert progress-alert-oneline closeable">
    <pre>one line alert</pre>
    <a class="closeable-close close">ok</a>
</div>

<!-- With SSE auto-connect -->
<form class="hidden progress-alert progress-alert-block closeable"
      data-async-progress-log="/job/log/url"
      data-async-target="main">
    <div class="log-target"></div>
</form>
.log-target = where JS appends <pre> lines
.closeable-close = click handler hides the entire alert
.alert-close-wrapper = hidden by default, revealed on close()
.error wrapper = hidden by default, revealed on error state via enableError()
data-async-progress-log = SSE endpoint URL for auto-connecting to server events
Reference: Live test page at /tests/progress_log · JS source: assets/src/js/lib/progressLog.js · CSS: @utility progress-alert in assets/src/styles/style.css · Test template: templates/views/tests/progress_log.html
